访问控制

三种访问限制符：public、protected、private。
定义在public之后的成员允许在整个程序内被访问
定义在private之后的成员可以被类的成员函数访问, 但不能在使用该类的代码处被访问
定义在protected之后的成员允许被子类的成员函数访问

静态成员和非静态成员

static 成员也有访问权限，可被修饰为public、private、protected。
static 成员在类内部声明，在类外部定义。
static 变量既能被静态函数访问，也能被非静态函数访问
non-static 变量只能被non-static function 访问

const修饰的成员函数

成员函数后加上const关键字，修饰的是this指针，也就是const对象也能调用该成员函数
常量版本的成员函数既可以被常量对象调用，又可以被非常量对象调用。
常量版本和非常量版本成员函数可以共存。（也就是可以当做重载）
共存时，常量对象会调用常量版本，非常量对象会调用非常量版本，并不会选择常量版本。

内联函数

内联函数可以避免函数调用时的开销
将函数指定为inline，则会在每个调用点内联展开，提高执行效率。
为函数加上关键字则只是向编译器提供建议，至于能不能变为内联函数具体要看编译器。
定义在类内部的成员函数是自动内联的。

操作符重载

操作符重载是类设计中较重要的环节，主要以下几种

  • 解引用（*) 成员访问运算符（->)
  • prefix or postfix increment and decrement （递增递减运算符）
  • 关系运算符（==、!=、<、<=、>、>=)
  • 算术运算符（+、+=、-、-=)
  • 赋值运算符（=）函数调用运算符（()）
  • 输入运算符（>>）输出运算符（<<）

  • 函数模板与普通函数在一起使用时的调用规则
    1. 普通函数可以隐式地转换类型，而函数模板会严格匹配类型
    2. 在既满足普通函数的调用，又满足函数模板的调用的情况下，编译器会优先选择调用普通函数
    3. 如果强制要选择函数模板，则可以使用空模板实参列表来限制 如sort<>(a,b);
    4. 模板函数支持重载（同普通函数重载的原则一样）
    5. 如果函数模板可以产生一个更好的匹配，则会选择模板
  • 函数模板本质剖析
    1. gcc编译器编译过程：预处理 ==>编译 ==> 汇编 ==>链接
    2. 编译器对函数模板的处理：
      (1) 编译器不会对函数模板编译产生满足要求的所有类型的函数
      (2)编译器会根据具体类型来编译不同的函数
      (3)编译器会进行二次编译
      第一次是在不确定类型的情况下对模板进行初步编译
      第二次是在调用函数时已经确定了类型时编译（同一种类型只编译一次）

## 四种类型强制转换

* **强制类型转换的格式**
  `cast-name<type>(expression)`
  `type`: 转换的目标类型, 如果为**引用**类型, 则转换结果为**左值**; `expression`: 要转换的值;
  `cast-name`: **static_cast**、**const_cast**、**dynamic_cast**、**reinterpret_cast**

* **static_cast**

> 只要不包含**底层const**，都可以使用**static_const**来强转类型
> 那什么是底层const呢？for example: `const char *p = a;` 这就是底层const，通过p指针不能修改a的值
> 那什么是顶层const呢？for example: `const int a = 0;` 这便是顶层const, a是常量，不可变更其值

```c++
// 避免损失精度, 可以请出static_cast
int i = 5, j = 3;
double slope = static_cast<double>(i) / j;

// 指针类型匹配
double a = 9.0;
void *p = &a; // void* 可以指向任何类型的变量
double *dp = static_cast<double*>(p); // 将void* 转换为double*, 不同于c语言（直接用`(double*)`强转）

  • dynamic_cast

有如下三种转换形式

dynamic_cast<type*>(e) // e必须是一个有效的指针, e的类型可以是type的公有派生类或者基类或者type
dynamic_cast<type&>(e) // e必须是一个左值
dynamic_cast<type&&>(e) // e必须是右值

类之间应该有继承关系，并且含有虚函数
第一种转换失败会返回0，第二种和第三种转换失败会抛出bad_cast异常

// 重载成员operator new or delete
class A {
public:
    A() { cout << "call ctor" << endl; }
    ~A() { cout << "call dtor" << endl; }
    void * operator new(size_t size) {
        cout << "A::operator new()" << endl;
        return ::operator new(size);
    }
    void * operator new[](size_t size) {
        cout << "A::operator new[]()" << endl;
        return ::operator new[](size);
    }
    void operator delete(void * p) {
        cout << "A::operator delete()" << endl;
        ::operator delete(p);
    }
    void  operator delete[](void *p) {
        cout << "A::operator delete[]()" << endl;
        ::operator delete[](p);
    }
};

void display_data_size() {
    cout << "指针大小: " << sizeof(char *) << endl;

    cout << "字符数据: " << endl;
    cout << "sizeof(char):\t\t\t" << sizeof(char) << endl;
    cout << "sizeof(unsigned char):\t\t" << sizeof(unsigned char) << endl;

    cout << "整型数据: " << endl;
    cout << "sizeof(short):\t\t\t" << sizeof(short) << endl;
    cout << "sizeof(unsigned short):\t\t" << sizeof(unsigned short) << endl;
    cout << "sizeof(int):\t\t\t" << sizeof(int) << endl;
    cout << "sizeof(unsigned int):\t\t" << sizeof(unsigned int) << endl;
    cout << "sizeof(long):\t\t\t" << sizeof(long) << endl;

    cout << "sizeof(double):\t\t\t" << sizeof(double) << endl;

}
